AI 时代前端代码写法指南

✅ 推荐写法（AI 友好）
1函数组件 · 单文件单职责
纯函数无类，每个组件独立文件。AI 每次读写范围有限，不会产生上下文污染。
2Tailwind 原子类
样式即文档，无运行时，无隐藏逻辑。AI 直接读 className 即可推断视觉效果。
3copy-in 组件（shadcn 模式）
组件源码在仓库里，AI 可直读直改。黑盒 npm 包内部逻辑 AI 无法触达。
4最小化 useEffect
副作用显式声明，不隐藏在库的内部生命周期中。AI 能完整追踪数据流。
5TypeScript 类型驱动
接口即契约，AI 看类型即知数据结构，减少猜测，提升修改精准度。
⚠️ 风险写法（AI 追加膨胀）
1Runtime CSS-in-JS（sx / emotion）
样式分散在 sx prop 里，AI 每次修改都倾向追加新属性，文件持续膨胀，难以审查。
2Vue SFC 大文件
script + template + style 三段合一。AI 生成时倾向在末尾追加，单文件行数快速增长。
3黑盒组件库深定制
MUI / AntD 内部逻辑 AI 无法修改，遇到限制只能堆 workaround，代码越来越脆。
4隐式全局状态
Context / Provide-Inject 跨层传递，AI 修改局部代码时难以感知全局影响范围。
